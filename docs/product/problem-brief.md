# Problem Brief (MVP)

유저 리서치를 먼저 크게 하지 않는다고 해도, "무엇을 검증할지"를 명확히 하기 위한 최소 문제 정의 문서.

Terminology SoT: [glossary.md](glossary.md)

## 타깃 세그먼트(초기)

- 유튜브로 무엇인가를 공부하는 사람 중, "노트로 지식을 추출/축적"하려는 기록형 학습자

## 상황(언제 이 문제가 터지나)

- 주로 데스크톱 환경
- 영상 타입에 따라 세션 길이가 달라짐
  - 강의/코스형: ~1시간
  - 데일리 큐레이션/요약형: ~20분

## AS-IS 워크플로우(현재/추정)

1. 유튜브에서 영상 탐색/재생
2. 노션/옵시디언/메모앱으로 이동
3. 링크 복붙 + 대충 타임스탬프 텍스트로 적기
4. 나중에 다시 찾기 어려움

## Frictions

- 산만함: 추천/연관/댓글/쇼츠로 이탈
- 기록 마찰: 앱 전환/컨텍스트 스위칭
- 레퍼런스 마찰: "이 노트가 영상 어디였지?"를 표현/회수하기 어려움

## Root Cause (가설)

- 유튜브는 소비 최적화(추천/체류)이고, 학습/회수 최적화가 아니다.
- 노트앱은 지식 관리 최적화이지만, 영상의 시간 축(근거)을 기본으로 다루지 않는다.

## 제품이 해결하려는 핵심(우선순위)

1. 기록 마찰: 시청과 기록이 분리되지 않게(한 화면)
2. 레퍼런스 마찰: 노트가 영상 "어느 지점"을 근거로 하는지 표현/회수(타임스탬프)
3. 산만함: 원하는 소스(채널/목록)만 보게 해서 학습 세션을 보호

## 문제 정의(한 문장)

- 유튜브 학습자는 영상의 특정 시점을 근거로 하는 노트를 빠르게 남기고, 그 근거를 다시 찾아 복습하기 어렵다.

## 성공 정의

- 유저가 학습 세션에서 타임스탬프 노트를 남기는 행동이 자연스럽게 발생한다.
- 같은 유저가 이후 세션에서도 반복적으로 타임스탬프 노트를 남기며(리텐션), 노트가 학습/복습 워크플로우에 남는다.

## 제약/리스크(초기)

- 유튜브 임베드/정책/쿼터에 영향을 받는다.
- 완전한 "영상 수집"보다 "학습 경험"에 집중해야 한다.
