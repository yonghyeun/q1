# Problem Brief (MVP)

유저 리서치를 먼저 크게 하지 않는다고 해도, "무엇을 검증할지"를 명확히 하기 위한 최소 문제 정의 문서.

## 타깃 세그먼트(초기)

- (예: 개발 강의/코스 기반 학습자)

## 상황(언제 이 문제가 터지나)

- (예: 퇴근 후 30~60분 학습 세션)

## AS-IS 워크플로우(현재/추정)

1. 유튜브에서 영상 탐색/재생
2. 노션/옵시디언/메모앱으로 이동
3. 링크 복붙 + 대충 타임스탬프 텍스트로 적기
4. 나중에 다시 찾기 어려움

## Frictions

- 산만함: 추천/연관/댓글/쇼츠로 이탈
- 기록 마찰: 앱 전환/컨텍스트 스위칭
- 레퍼런스 마찰: "이 노트가 영상 어디였지?"를 표현/회수하기 어려움

## Root Cause (가설)

- 유튜브는 소비 최적화(추천/체류)이고, 학습/회수 최적화가 아니다.
- 노트앱은 지식 관리 최적화이지만, 영상의 시간 축(근거)을 기본으로 다루지 않는다.

## 제품이 해결하려는 핵심(우선순위)

- (예: 3번 레퍼런싱 부재를 먼저 해결한다.)

## 문제 정의(한 문장)

- (예: 유튜브 학습자는 "근거가 남는 노트"를 남기기 어려워 지식이 축적되지 않는다.)

## 성공 정의

- (예: 기록이 쌓이고, 다시 찾고, 다시 재생으로 이어지는 루프가 생긴다.)

## 제약/리스크(초기)

- 유튜브 임베드/정책/쿼터에 영향을 받는다.
- 완전한 "영상 수집"보다 "학습 경험"에 집중해야 한다.
