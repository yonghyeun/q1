# One-Pager (MVP)

작게 시작하기 위한 1장 요약. 길게 쓰지 않는다.

## 제품명(가제)

- (예: YouTube Study Notes)

## 타깃(한 문장)

- (예: 유튜브로 강의/코스를 공부하는 사람이, 복습 가능한 노트를 빠르게 남기고 다시 찾게 만든다.)

## 문제(왜 지금 불편한가)

- 산만함: 추천/연관/쇼츠로 이탈한다.
- 분리: 시청과 기록이 분리돼 맥이 끊긴다.
- 근거 부재: 기록이 영상의 어느 지점을 근거로 하는지 남기기 어렵다.

## 해결(핵심 경험 1줄)

- (예: 영상 재생과 마크다운 노트를 한 화면에서, 단축키로 타임스탬프를 남기고 클릭으로 다시 점프한다.)

## 차별점(딱 1~2개)

- (예: 타임스탬프 기반 레퍼런싱이 "기본"이다.)
- (예: 큐레이션된 소스만 보여주는 집중 학습 환경)

## MVP 범위(In Scope)

- (예: 채널/플레이리스트 URL 등록)
- (예: 영상 목록 + 임베드 플레이어)
- (예: 마크다운 노트 + 타임스탬프 삽입/클릭 점프)
- (예: 최소 검색)

## Out of Scope (이번 사이클에 안 함)

- (예: 영상 다운로드/편집/클리핑)
- (예: 복잡한 팀 협업 기능)
- (예: 완성도 높은 추천 알고리즘)

## 성공 지표(초기)

- Activation: (예: 첫 세션에서 타임스탬프 삽입 1회 이상 비율)
- Retention: (예: D7 재방문)

## 오픈 질문

- (예: 채널 기반이냐, 플레이리스트 기반이냐, 혹은 "학습 목록" UX냐?)
- (예: 어떤 학습 도메인부터 공략할지?)
