# DEC: Minimize "living docs" for zero-to-one (deploy first)

- Date: 2026-03-03
- Context: MVP를 빠르게 배포(Deploy)하며 0→1을 만드는 단계에서는 문서의 완성도보다 배포 속도와 실행력이 중요하다. 문서가 여러 파일로 분산되면 관리 비용이 커져 개발/배포 루프가 느려질 수 있다.
- Decision: zero-to-one 단계에서는 "살아있는 문서"를 최소화한다. 일상적으로 업데이트하는 문서는 `docs/product/mvp-spec.md`를 중심으로 1~2개로 제한하고, 나머지 문서는 템플릿/참조로 유지하되 필요할 때만 갱신한다. `context/decisions/`는 폴더 구조/스코프 컷/정본 위치 같은 큰 결정에만 사용한다.
- Alternatives: 모든 정의/로그 문서를 매 루프마다 업데이트한다. 모든 문서를 하나로 합쳐 운영한다.
- Tradeoffs: 일부 정보가 여러 파일에 흩어져 있을 수 있고, 참조 문서는 최신성이 떨어질 수 있다. 대신 문서 관리가 배포의 병목이 되는 것을 막고, 0→1 단계의 실행 속도를 확보한다.
- Revisit if: 제품이 안정화되어 실험/계측/협업의 비중이 커지거나, 문서 부정합으로 재작업이 반복될 때(이때 living docs 범위를 확대하거나 구조를 재정렬한다).
