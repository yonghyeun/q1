# DEC: Add escape hatches beyond "latest 30" (pagination + video URL)

- Date: 2026-03-04
- Context: unote는 유튜브로 공부하는 유저가 "시청 + 타임스탬프 노트"를 한 화면에서 빠르게 반복하게 만드는 것이 핵심이다. 초기 MVP를 "채널 등록 + 최신 30개"로만 제한하면, (1) 학습이 최신 업로드 소비가 아니라 과거 강의/시리즈를 따라가는 경우가 많고 (2) 유저가 찾고 싶은 특정 영상을 결국 유튜브에서 다시 검색하게 되어 산만함/이탈이 재발할 수 있다.
- Decision: 기본 목록은 "큐레이션된 채널들의 최신 N개(기본 30)"로 유지하되, 학습/노트테이킹을 망치지 않기 위한 탈출구 2가지를 MVP에 포함한다.
  - Pagination(전체 기준): "Load older"로 30개씩 추가 로딩한다. 기본은 전체(모든 등록 채널을 합친 목록)이며, 웹앱 내부에서 채널 필터로 원하는 소스만 좁혀볼 수 있게 한다. `Load older`는 "현재 선택된 필터" 기준으로 동작하며, `All` 피드는 채널별 버퍼를 `publishedAt` 기준으로 머지해 가상 페이지네이션을 만든다(필요한 채널만 추가 fetch).
  - Video URL Open: 큐레이션된 채널 외에도 유저가 유튜브 "영상 URL"을 붙여넣으면 바로 Player+Note 화면으로 진입해 노트테이킹을 할 수 있게 한다.
  - Playlist(다음 단계): 플레이리스트 URL/ID 기반 큐레이션은 이번 사이클에서는 구현하지 않되, 코어 UX 검증 이후 "다음 단계(MVP 후속)"에서 구현할 항목으로 명시한다.
- Alternatives: 최신 30개만 유지한다. 플레이리스트 지원을 지금 당장 포함한다. 채널당 페이지네이션만 제공한다(통합 목록 없음).
- Tradeoffs: 목록/로딩/필터 UX와 데이터 가져오기 로직이 복잡해질 수 있다. "큐레이션된 소스만"이라는 제약이 Video URL Open으로 느슨해질 수 있다. 대신 유저가 유튜브로 되돌아가 검색/탐색하다 이탈하는 문제를 줄이고, 학습 세션에서의 실사용 가능성을 높인다.
- Revisit if: API/쿼터 또는 로딩 성능 문제가 병목이 되거나, 유저가 채널 목록보다 Video URL Open만 사용한다면(온보딩/홈 플로우 재설계 필요). 플레이리스트 수요가 반복적으로 발생한다면(playlist 소스 우선순위 상향).
