# DEC: Separate `context/` and `docs/` (MVP)

- Date: 2026-03-03
- Context: MVP를 `deploy -> 계측 -> 가설 검증 -> 수정` 루프로 빠르게 반복하는 중이며, 문서가 늘어날수록 목적/수명/변경 빈도가 다른 문서들이 섞여 관리 비용이 증가할 위험이 있다.
- Decision: 실행/운영/실험에 직접 쓰이는 문서는 `context/`에, 설명/공유 목적의 제품 문서는 `docs/`에 분리해 관리한다.
- Alternatives: 모든 문서를 `docs/` 아래에 통합한다. 모든 문서를 `context/` 아래에 통합한다.
- Tradeoffs: 문서가 두 군데로 나뉘어 "어디에 써야 하는지" 판단이 필요하다. 대신 목적별로 문서가 정돈되고, 실험 로그가 커져도 제품 문서가 오염되지 않는다.
- Revisit if: 팀/협업이 커져 별도 문서 사이트(MkDocs 등)나 ADR 체계가 필요해질 때, 또는 `context/`가 너무 커져 실행 입력값으로서의 장점이 사라질 때.
