# Agent Team

이 디렉토리는 이 저장소에서 사용할 에이전트 팀 운영 방식을 관리한다.
제품 기능 문서나 프로젝트 도메인 문서가 아니라, 작업 방식과 협업 구조를 다룬다.

## Boundary
- `agent-team/`은 에이전트 팀의 목표, 역할, 경계, handoff, 승인 체계를 다룬다.
- `context/`, `docs/`, `policies/`는 저장소와 제품 자체의 운영 문서를 유지한다.
- backlog 입력의 기본 SoT는 GitHub issue를 따른다.
- 같은 내용을 두 위치에 중복 기록하지 않는다.
- 프로젝트 문서와 연결이 필요하면 복사 대신 링크로 연결한다.

## Working Model
- 멀티에이전트 도입은 JIT 방식으로 진행한다.
- 합의된 단계만 문서화한다.
- 순서는 `roadmap.md`를 따른다.
- 규칙으로 굳지 않은 내용은 성급히 policy로 승격하지 않는다.

## Growth Rule
- 현재 필요한 문서만 추가한다.
- 반복되는 산출물이 생기면 그때 하위 디렉토리를 만든다.
- 산출물은 가능한 한 원자적 작업 단위로 관리한다.
