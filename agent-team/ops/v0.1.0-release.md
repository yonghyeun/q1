# Agent Team 운영 통합 구성 v0.1.0

## 핵심 변경
- 운영 기준을 `task/agent 중심`에서 `trace/span 중심`으로 확장
- `task-brief.json` 이후 실행은 `trace.md` 기반 수동 span 운영
- span 실행 근거를 `run-log.md`에 기록하고, KPI는 `run-report.json`으로 집계

## 필수 운영 아티팩트
- `task-brief.json`
- `trace.md`
- `run-log.md`
- `run-report.json` (종료/판정 시)
- `status.md`

## 운영 원칙
1. 모든 span은 human decision이 있어야 종료된다.
2. run-log 필수 항목 누락 시 다음 span으로 이동하지 않는다.
3. 실패 2회 누적 시 process/codebase 우선 개선안을 `feedback-record.json`에 기록한다.

## 파일럿
- 파일럿: 운영팀이 지정한 task-id로 1회 수행
- 목표: v0.1.0 수동 trace 운영 1회 검증
- 완료 조건:
  - span 로그 누락 0
  - human decision 누락 0
  - run-log와 run-report 수치 일치
