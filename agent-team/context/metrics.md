# Metrics Definition

## Why These Metrics
- 초기 목표는 빨리 처리하는 것보다 끝까지 안정적으로 완주하는 것이다.
- 따라서 속도 지표보다 성공률과 재발률 지표를 먼저 본다.
- 리드타임은 버리지 않되, 초기에 팀을 왜곡시키지 않도록 보조 지표로 둔다.

## Core Metrics
### 1. 전체 trace 성공률
- 의미: 하나의 작업이 전체 실행 흐름을 끝까지 완료한 비율이다.
- trace 범위: 원자 분해 -> 계획 -> 작업 -> 로깅 -> 검증.
- 왜 중요한가: "결국 목표를 달성했는가"를 가장 직접적으로 보여준다.
- 이렇게 읽는다: 값이 낮으면 전체 흐름 어딘가에서 자주 끊긴다는 뜻이다.
- 주의점: 성공/실패 판정 기준이 흐리면 수치가 의미를 잃는다.

### 2. node별 성공률
- 의미: trace를 이루는 각 단계별 성공 비율이다.
- node 예시: 분해, 계획, 작업, 로깅, 검증.
- 왜 중요한가: 어디가 병목인지 위치를 찾을 수 있다.
- 이렇게 읽는다: 전체 성공률은 괜찮아도 특정 node 성공률이 낮으면 그 단계가 약점이다.
- 주의점: node 정의를 자주 바꾸면 비교가 어려워진다.

### 3. 동일 병목 재발률
- 의미: 한 번 수정한 병목이 이후 실행에서 다시 나타나는 비율이다.
- 왜 중요한가: 피드백 루프가 실제 개선으로 이어지는지 확인할 수 있다.
- 이렇게 읽는다: 값이 높으면 문제를 기록만 하고 해결하지 못한다는 뜻이다.
- 주의점: 병목 분류 기준이 불명확하면 재발 판정도 흔들린다.

## Supporting Metrics
### 리드타임
- 의미: 하나의 작업이 시작부터 종료까지 걸린 시간이다.
- 위치: 보조 관측 지표.
- 이유: 중요하지만, 초반에는 속도 최적화가 품질 저하를 부를 수 있다.
- 활용: 구조가 안정화된 뒤 효율성 개선 단계에서 비중을 높인다.

## Reading Guide
- 전체 trace 성공률이 낮다.
  - 전체 흐름 자체가 불안정하다는 뜻이다.
- node별 성공률 중 특정 단계만 낮다.
  - 그 단계가 우선 개선 대상이다.
- 동일 병목 재발률이 높다.
  - 개선 조치가 약하거나, 로그와 분류가 부정확하다는 뜻이다.
- 리드타임만 낮아진다.
  - 빨라졌다고 좋아할 수 없다.
  - 성공률과 재발률이 같이 유지되는지 확인해야 한다.
