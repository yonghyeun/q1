# Goal Definition

## Goal
- 입력은 WBS 또는 해야 할 작업 목록이다.
- agent-team은 이를 원자적 작업 단위로 분해하고, 계획, 작업, 로깅, 검증, 병목 탐지와 수정의 피드백 루프로 처리해야 한다.
- 목표는 단순 업무 자동화가 아니라, 측정 가능하고 개선 가능한 멀티에이전트 개발 하네스를 만드는 것이다.

## In Scope
- 작업을 원자 단위로 분해하는 흐름
- 분해 결과를 바탕으로 계획을 수립하는 흐름
- 실제 작업 실행과 산출물 생성 흐름
- 실행 로그와 근거를 남기는 흐름
- 결과 검증과 병목 식별 흐름
- 병목 수정 후 다음 실행에 반영하는 피드백 루프

## Out Of Scope
- 이슈 생성, PR 생성 자체를 1차 목표로 두지 않는다.
- 해당 영역은 이미 준비된 skill을 활용한다.

## Success Criteria
- 성공 기준은 프로젝트 goal 달성에 기여하는지 여부다.
- 초기 우선순위는 속도보다 품질이다.
- 초기 우선순위는 속도보다 안정성이다.
- 초기 우선순위는 속도보다 안정적 운영 가능성이다.
